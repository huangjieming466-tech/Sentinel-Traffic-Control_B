# 双路智能交通灯控制系统 (Dual-Camera Smart Traffic Control System)

本项目基于 Orange Pi 5 (RK3588) 硬件平台，利用计算机视觉技术实现了一个双路智能交通灯控制系统。系统通过双 USB 摄像头实时监测两个路口的交通流量，并基于 YOLOv8 模型进行车辆检测，根据排队情况自适应调整红绿灯时长，以最大化路口通行效率。

## 🛠 硬件环境

*   **计算平台**: Orange Pi 5 (RK3588 芯片)
*   **操作系统**: Ubuntu (Linux)
*   **视频输入**: 2个 USB 摄像头
*   **显示输出**: HDMI 显示器 (用于显示实时监控画面和状态)

## 🏗 技术路线

1.  **视频采集**:
    *   使用 OpenCV 读取两个 USB 摄像头的视频流。
    *   **优化策略**: 强制使用 `MJPG` 编码格式和 `640x640` 分辨率，以降低 USB 带宽占用并提高帧率。
2.  **AI 推理 (Inference)**:
    *   **模型**: YOLOv8 Nano (最轻量级版本)。
    *   **加速**: 支持 NCNN 导出模型以在端侧设备上获得更高推理速度。
    *   **目标检测**: 仅检测车辆相关类别 (Car, Bus, Truck, Motorcycle)。
3.  **逻辑控制**:
    *   基于 **有限状态机 (Finite State Machine, FSM)** 实现交通灯状态流转。
    *   基于实时车辆计数比率进行动态调度。
4.  **可视化**:
    *   实时绘制检测框、车辆统计数据、模拟红绿灯信号及倒计时。
    *   双路画面拼接显示。

## 🚦 控制逻辑与交通规则

系统核心采用自适应调度算法，不再使用固定的红绿灯时长，而是根据实时路况动态调整。

### 1. 状态定义
系统包含四个循环状态：
*   **GREEN_A**: A路绿灯，B路红灯 (A路通行)
*   **YELLOW_A**: A路黄灯，B路红灯 (A路清空)
*   **GREEN_B**: B路绿灯，A路红灯 (B路通行)
*   **YELLOW_B**: B路黄灯，A路红灯 (B路清空)

此外，为了行车安全，系统增加了 **全红时段 (All-Red Clearance)**：
*   **ALL_RED**: 每次黄灯切换到绿灯前，会有 **2秒** 的时间两边都是红灯，用于清空路口，防止左转车辆相撞。

### 2. 时间参数
*   `MIN_GREEN_TIME` (最小绿灯时间): **5秒** - 保证车辆启动和基本通行时间。
*   `MAX_GREEN_TIME` (最大绿灯时间): **30秒** - 防止单向长时间占用，保证公平性。
*   `YELLOW_TIME` (黄灯时间): **3秒** - 状态切换缓冲。
*   `ALL_RED_TIME` (全红时间): **2秒** - 绝对安全时间。

### 3. 智能调度算法
在 **绿灯** 状态期间，系统每帧检测并计算 A、B 两路口的车辆数量 (`count_A`, `count_B`)。

**切换规则**:
当当前绿灯持续时间超过 `MIN_GREEN_TIME` 后，若满足以下任一条件，立即切换到黄灯状态：

1.  **超时切换**: 当前绿灯时间已达到 `MAX_GREEN_TIME` (30秒)。
2.  **拥堵优先切换**:
    *   **基础条件**: 红灯方向的等待车辆数是绿灯方向通行车辆数的 **1.5倍** (`count_waiting > count_passing * 1.5`)。
    *   **低流量阈值**: 且红灯方向等待车辆数 **大于3辆** (`count_waiting > 3`)，防止低流量时频繁切换。
    *   **去抖动 (Debounce)**: 上述两个条件必须 **连续满足 3秒钟**，系统才会真正执行切换。这有效防止了因 AI 识别抖动导致的红绿灯乱跳。
    *   **“耐心值”权重 (Weighted Scoring)**: 引入公平算法，解决单辆车长时间等待问题。
        *   **公式**: `Score = Vehicle Count + (Red Light Wait Time / 10)`
        *   即使只有一辆车，随着等待时间增加，其分数会逐渐升高，最终触发切换。

**逻辑图示**:
```
[GREEN_A] --(超时 OR B路拥堵)--> [YELLOW_A] --> [GREEN_B] --(超时 OR A路拥堵)--> [YELLOW_B] --> [GREEN_A] ...
```

## 📂 文件说明

*   `smart_traffic.py`: **主程序**。包含双路视频采集、YOLOv8 推理、状态机控制逻辑及 UI 绘制。
*   `ai_detect.py`: **单路测试脚本**。用于测试单摄像头、分辨率设置及简单的 ROI 区域检测功能。
*   `benchmark.py`: **性能测试脚本**。不打开摄像头，仅测试 NPU/CPU 运行 YOLOv8n 模型的推理速度 (FPS)。

## 🚀 快速开始

### 1. 环境依赖安装
确保已安装 Python 3 和必要的库：
```bash
pip install opencv-python ultralytics numpy
```

### 2. 运行性能测试
检查硬件性能是否达标：
```bash
python3 benchmark.py
```

### 3. 启动智能交通系统
连接两个 USB 摄像头后运行：
```bash
python3 smart_traffic.py
```

## ⚠️ 注意事项
*   **摄像头索引**: 程序默认尝试读取 index `0` 和 `2`。如果无法打开第二个摄像头，请修改 `smart_traffic.py` 中的 `init_camera([2, 1])` 参数。
*   **光照影响**: 夜间或强逆光可能会影响车辆检测准确率，建议根据实际场景调整摄像头角度。
